```sql
creator_id  UUID  PRIMARY KEY
access_token  TEXT  NOT NULL

```

#### OAuth フロー

```
/settings/calendar
  → /api/auth/google          (認証チェック → Google 認可 URL にリダイレクト)
  → Google 同意画面
  → /api/auth/google/callback (token 取得 → creator_tokens に保存)
  → /settings/calendar?cal=connected
```

エラー時は全て `/settings/calendar?cal=error&msg=...` にリダイレクト（raw JSON を返さない）。

## バグ修正履歴

### 1. Google OAuth `invalid_client` エラー

**原因**: `.env.local` の `GOOGLE_CLIENT_SECRET` が Google Cloud Console のものと不一致。  
**修正**: Google Cloud Console からクライアントシークレットを再取得して `.env.local` を更新。  
**コード修正**: OAuth コールバックのエラーハンドリングを強化し、エラー内容を UI に表示するよう変更。

```ts
// app/api/auth/google/callback/route.ts
const oauthError = req.nextUrl.searchParams.get('error')
if (oauthError) {
  const msg = encodeURIComponent(oauthError === 'access_denied' ? 'カレンダー連携をキャンセルしました' : `Google認証エラー: ${oauthError}`)
  return NextResponse.redirect(`${process.env.NEXT_PUBLIC_APP_URL}/settings/calendar?cal=error&msg=${msg}`)
}
```

### 2. `working_days_required=0` でクラッシュ

**原因**: 0 や負の値が渡されると `calculateDeadline` が `workingDays` が空のまま終了し、`deadline` が `undefined` になり `.getFullYear()` でクラッシュ。  
**修正**:

- `app/api/deadline/suggest/route.ts` にバリデーション追加（`< 1` または `> 90` → 400）
- `lib/calculateDeadline.ts` に空配列ガード追加

```ts
// lib/calculateDeadline.ts
if (workingDays.length === 0) {
  throw new Error('有効な稼働日が見つかりませんでした（作業日数が0以下です）')
}
```

### 3. `order_type` カラム未作成による 42703 エラー

**原因**: `supabase/005_order_type.sql` を本番 DB に未適用の状態で INSERT が失敗。  
**修正**: `app/api/orders/route.ts` にフォールバックを実装（`42703` エラー時に `order_type` を除外して再試行）。

```ts
if (error?.code === '42703') {
  const { order_type: _removed, ...payloadWithoutType } = insertPayload;
  ({ data: order, error } = await db.from('projects').insert(payloadWithoutType).select('id').single())
}
```

### 4. `schedule.days` の型不一致

**原因**: DB には数値配列 `[1,2,3,4,5]` が保存されているのに、コードが日本語曜日名として処理しようとしていた。  
**修正**: 型チェックで分岐するよう修正（`typeof rawDays[0] === 'number'` を優先）。

### 5. `order.order_type` の TypeScript 型エラー

**原因**: SELECT クエリに `order_type` を含めていないため、TypeScript が型を認識しない。  
**修正**: `(order as Record<string, unknown>).order_type` キャストで対応。

## 注意事項・パターン

- **OAuth エラーは必ず UI にリダイレクト** — `?cal=error&msg=...` 形式。raw JSON を返さない。
- **マイグレーション安全パターン** — DB 変更前後でコードが動作するようフォールバックを実装。
- **`creator_tokens` は service role クライアントでアクセス** — RLS がないためクライアント側からは直接アクセス不可。
- **`schedule.days` は数値配列** — DB の保存形式は `[1,2,3,4,5]`（0=日, 6=土）。文字列配列のフォールバックも実装済み。
