## DB スキーマ（Supabase）

```
supabase/
  002_full_schema.sql   — 全テーブル定義 + RLS ポリシー
  003_auth_trigger.sql  — 新規ユーザー時に public.users を自動作成
  004_project_boards.sql — project_boards / project_roles / project_tasks
```

### 主要テーブル
| テーブル | 用途 |
|---|---|
| `users` | auth.users の拡張（roles, display_name, sns_links, entity_type） |
| `creator_profiles` | クリエイター詳細（skills, creator_type, bio, price_min, availability） |
| `portfolios` | ポートフォリオ URL（platform, url, title, thumbnail_url） |
| `project_boards` | プロジェクトボード（title, category, status, is_public） |
| `project_roles` | 役職（role_name, is_owner_role, assigned_user_id） |
| `project_tasks` | タスク（title, status: todo/in_progress/done, role_id, due_date） |
| `projects` | 依頼（client_id, creator_id, budget, deadline, status） |

## 注意事項

- **`export const dynamic = 'force-dynamic'`** はデータを毎回 DB から取得するサーバーコンポーネントに必須
  （Next.js のキャッシュで古いデータが返るバグを防ぐ）
- **`.single()` は使わない** → `.limit(1)` + `[0] ?? null` パターンを統一
  （行が 0 件のとき `.single()` は 406 エラーをスローする）
- **JSONB 配列カラム**（`sns_links`, `skills`, `creator_type`）は `Array.isArray()` ガード必須
  （Supabase は空配列を `{}` として返すことがある）
- **サーバーコンポーネントにイベントハンドラ不可**
  （`onMouseEnter`, `onClick` 等は `'use client'` コンポーネントのみ）
